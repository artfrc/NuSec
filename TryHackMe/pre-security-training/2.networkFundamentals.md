# 🌐 Fundamentos de Redes

## 🇬🇧 English

### 🔍 Device Identification on a Network
Devices on a network are identified using:
- **IP Address** 📍
- **MAC Address (Media Access Control)** 🔗

---

### 📡 IP Address

#### Structure
An IP address consists of four octets, each ranging from 0 to 255:
```
Octet 1   Octet 2   Octet 3   Octet 4
192.      168.      1.        1
0–255     0–255     0–255     0–255
```

- IP addresses are **not globally unique** and can change.
- Within a single network, **no two devices can have the same IP address** simultaneously.
- Devices may have **public** or **private** IP addresses depending on their network location.

#### Example Devices
| Device Name       | IP Address     | IP Type  |
|-------------------|----------------|----------|
| DESKTOP-KJE57FD   | 192.168.1.77   | Private  |
| DESKTOP-KJE57FD   | 86.157.52.21   | Public   |
| CMNatic-PC        | 192.168.1.74   | Private  |
| CMNatic-PC        | 86.157.52.21   | Public   |

**MAC Addresses**:
- DESKTOP-KJE57FD: `EC:5C:68:C3:7E:51`
- CMNatic-PC: `50:3E:AA:E8:3B:64`

#### IPv4 vs IPv6
- **IPv4**: Supports ~4.29 billion addresses (2³²).
- **IPv6**: Supports 340 trillion+ addresses (2¹²⁸).

**Examples**:
- IPv4: `86.157.52.21`
- IPv6: `2a00:22c4:a531:c500:425f:cce6:c36b:f64d`

---

### 🔗 MAC Address
- Each **Network Interface Card (NIC)** has a **unique MAC address**.
- Format: `a4:c3:f0:85:ac:2d`
  - First 6 characters: Manufacturer 🏭
  - Last 6 characters: Unique ID 🆔

#### MAC Spoofing
- MAC addresses can be **faked (spoofed)** 🎭.
- This can **bypass security** if access control relies solely on MAC addresses.

---

### 📬 Ping and ICMP
- The `ping` command uses **ICMP packets** to test network connectivity 📡.
- ICMP sends an **echo request** and expects an **echo reply** to measure latency ⏱️.

---

### 🌐 LAN Topologies

#### Star Topology ⭐
- Devices connect via a **central switch/hub**.
- **Reliable and scalable**, but expensive 💸.
- Failure of the central device disrupts all communication 🚫.

#### Bus Topology 🚌
- Devices share a **single backbone cable**.
- **Cheap**, but prone to **bottlenecks** and **single point of failure** ⚠️.

#### Ring Topology 🔄
- Devices are connected in a **loop**.
- Requires **fewer cables** than star topology.
- A single cable failure disrupts the **entire network** 🚨.

---

### 🌐 Subnetting
Subnetting divides a network into **smaller subnetworks** 🛠️.

#### Components:
- **Network Address**: e.g., `192.168.1.0`
- **Host Address**: e.g., `192.168.1.1`
- **Default Gateway**: e.g., `.1` or `.254`, used for routing external traffic 🚦.

---

### 🔄 ARP (Address Resolution Protocol)
- Maps **IP addresses to MAC addresses** 🔗.
- Devices maintain an **ARP cache** 🗃️.

#### ARP Workflow:
1. **ARP Request**: "Who has this IP?" 📢
2. **ARP Reply**: "I do! Here’s my MAC address." 🗣️

---

### 🤝 DHCP (Dynamic Host Configuration Protocol)
- Automatically assigns **IP addresses** to devices 📡.

#### Process:
1. **DHCP Discover**: "Can anyone give me an IP?" 🔍
2. **DHCP Offer**: "Here’s `192.168.1.10`." 📬
3. **DHCP Request**: "I’ll take it!" ✅
4. **DHCP Acknowledgment**: "Good for 24 hours." ⏳

---

### 📚 OSI Model
The OSI model defines seven layers for network communication:

1. **Physical** ⚡️: Hardware transmission (e.g., Ethernet).
2. **Data Link** 🔗: MAC addressing and data formatting.
3. **Network** 🌐: Routing using IP addresses. Protocols: OSPF, RIP.
4. **Transport** 🚚: Ensures data delivery.
   - **TCP**: Reliable, slower 🐢.
   - **UDP**: Fast, no guarantee 🏃.
5. **Session** 🤝: Manages sessions and checkpoints.
6. **Presentation** 🎨: Data formatting and translation.
7. **Application** 📱: User-level protocols and interfaces.

---

### 📝 Summary
- **IP and MAC addresses** are critical for network communication.
- **LAN topologies** impact network design, cost, and reliability.
- Protocols like **ARP**, **DHCP**, and **ICMP** enable device discovery and communication.
- The **OSI model** provides a standard framework for data exchange across networks.

## 🇧🇷 Português

### 🔍 Identificação de Dispositivos em uma Rede
Os dispositivos em uma rede são identificados por:
- **Endereço IP** 📍
- **Endereço MAC (Media Access Control)** 🔗

---

### 📡 Endereço IP

#### Estrutura
Um endereço IP é composto por quatro octetos, cada um variando de 0 a 255:
```
Octeto 1  Octeto 2  Octeto 3  Octeto 4
192.      168.      1.        1
0–255     0–255     0–255     0–255
```

- Endereços IP **não são globalmente únicos** e podem mudar.
- Em uma única rede, **dois dispositivos não podem ter o mesmo endereço IP** simultaneamente.
- Dispositivos podem ter endereços IP **públicos** ou **privados**, dependendo de sua localização na rede.

#### Exemplos de Dispositivos
| Nome do Dispositivo | Endereço IP    | Tipo de IP |
|---------------------|----------------|------------|
| DESKTOP-KJE57FD     | 192.168.1.77   | Privado    |
| DESKTOP-KJE57FD     | 86.157.52.21   | Público    |
| CMNatic-PC          | 192.168.1.74   | Privado    |
| CMNatic-PC          | 86.157.52.21   | Público    |

**Endereços MAC**:
- DESKTOP-KJE57FD: `EC:5C:68:C3:7E:51`
- CMNatic-PC: `50:3E:AA:E8:3B:64`

#### IPv4 vs IPv6
- **IPv4**: Suporta ~4,29 bilhões de endereços (2³²).
- **IPv6**: Suporta mais de 340 trilhões de endereços (2¹²⁸).

**Exemplos**:
- IPv4: `86.157.52.21`
- IPv6: `2a00:22c4:a531:c500:425f:cce6:c36b:f64d`

---

### 🔗 Endereço MAC
- Cada **Placa de Interface de Rede (NIC)** possui um **endereço MAC único**.
- Formato: `a4:c3:f0:85:ac:2d`
  - Primeiros 6 caracteres: Fabricante 🏭
  - Últimos 6 caracteres: ID único 🆔

#### Spoofing de MAC
- Endereços MAC podem ser **falsificados (spoofing)** 🎭.
- Isso pode **burlar a segurança** se o controle de acesso for baseado apenas em endereços MAC.

---

### 📬 Ping e ICMP
- O comando `ping` utiliza **pacotes ICMP** para testar a conectividade de rede 📡.
- O ICMP envia uma **solicitação de eco** e espera uma **resposta de eco** para medir a latência ⏱️.

---

### 🌐 Topologias de LAN

#### Topologia Estrela ⭐
- Dispositivos conectam-se por meio de um **switch/hub central**.
- **Confiável e escalável**, mas caro 💸.
- Falha do dispositivo central interrompe toda a comunicação 🚫.

#### Topologia Barramento 🚌
- Dispositivos compartilham um **cabo backbone único**.
- **Barato**, mas propenso a **gargalos** e **ponto único de falha** ⚠️.

#### Topologia Anel 🔄
- Dispositivos são conectados em um **loop**.
- Requer **menos cabos** que a topologia estrela.
- A falha de um cabo interrompe **toda a rede** 🚨.

---

### 🌐 Subnetting
Subnetting divide uma rede em **sub-redes menores** 🛠️.

#### Componentes:
- **Endereço de Rede**: e.g., `192.168.1.0`
- **Endereço de Host**: e.g., `192.168.1.1`
- **Gateway Padrão**: e.g., `.1` ou `.254`, usado para rotear tráfego externo 🚦.

---

### 🔄 ARP (Protocolo de Resolução de Endereços)
- Mapeia **endereços IP para endereços MAC** 🔗.
- Dispositivos mantêm um **cache ARP** 🗃️.

#### Fluxo de Trabalho do ARP:
1. **Solicitação ARP**: "Quem tem este IP?" 📢
2. **Resposta ARP**: "Eu tenho! Aqui está meu endereço MAC." 🗣️

---

### 🤝 DHCP (Protocolo de Configuração Dinâmica de Host)
- Atribui **endereços IP automaticamente** 📡.

#### Processo:
1. **DHCP Discover**: "Alguém pode me dar um IP?" 🔍
2. **DHCP Offer**: "Aqui está `192.168.1.10`." 📬
3. **DHCP Request**: "Eu aceito!" ✅
4. **DHCP Acknowledgment**: "Válido por 24 horas." ⏳

---

### 📚 Modelo OSI
O modelo OSI define sete camadas para comunicação de rede:

1. **Física** ⚡️: Transmissão de hardware (e.g., Ethernet).
2. **Enlace de Dados** 🔗: Endereçamento MAC e formatação de dados.
3. **Rede** 🌐: Roteamento usando endereços IP. Protocolos: OSPF, RIP.
4. **Transporte** 🚚: Garante a entrega de dados.
   - **TCP**: Confiável, mais lento 🐢.
   - **UDP**: Rápido, sem garantia 🏃.
5. **Sessão** 🤝: Gerencia sessões e pontos de verificação.
6. **Apresentação** 🎨: Formatação e tradução de dados.
7. **Aplicação** 📱: Protocolos e interfaces no nível do usuário.

---

### 📝 Resumo
- **Endereços IP e MAC** são essenciais para a comunicação em rede.
- **Topologias de LAN** afetam o design, custo e confiabilidade da rede.
- Protocolos como **ARP**, **DHCP** e **ICMP** permitem a descoberta e comunicação de dispositivos.
- O **modelo OSI** fornece uma estrutura padrão para troca de dados em redes.
